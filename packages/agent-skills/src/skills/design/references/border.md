# Border

Borders explain separation, containment, focus, and state. Use border as a full mode when edge language is the request; when loaded as support, change only edges touched by the active mode.

## Give every edge a job

A border may:

- Separate sections or close visual neighbors
- Contain a proof object, image, form, or interactive demo
- Define a control
- Show focus, selection, error, or disabled state
- Establish the page's physical or editorial frame language

If spacing, alignment, or surface contrast already separates content, another line is unnecessary.

## Build a small edge vocabulary

Use a limited set of weights and semantic colors. Most structural borders should remain quiet. Focus and selected states can be stronger because they carry behavior.

Avoid arbitrary one-off opacity values and unrelated border colors. On dark surfaces, author edge values for the actual background rather than inverting light-theme lines.

## Radius is physical language

Radius should support the art direction: precise, soft, tactile, formal, raw, or technical. Keep nested radii visually related; inner corners usually need less radius than outer containers.

Pills belong to controls or brand systems that genuinely use them. Mixing square frames, medium cards, and full pills without a clear hierarchy makes the page feel assembled.

## Focus and controls

Focus treatment is the most important edge. It must be visible, offset or inset without clipping, and consistent across links, buttons, form fields, menus, and disclosures that exist.

Inputs need clear resting, focus, invalid, valid when useful, disabled, and read-only boundaries only for states they can enter. Do not rely on border color alone for errors or selection.

## Dividers and frames

Use dividers between thoughts that spacing alone cannot separate. Editorial rules can create pace; technical frames can create rigor; image windows can support proof. Repeated card outlines should not substitute for hierarchy.

Border and shadow can coexist, but one should lead. A strong line plus a strong shadow usually creates noise.

## Refuse

- Accent side rails as automatic decoration
- Borders around every section or text group
- Random radius mixing
- Focus outlines removed without replacement
- Heavy borders everywhere
- Double boundaries where nested elements meet
- Product-table edge systems on pages with no table or embedded demo

## Verification

Use screenshots to judge separation, physical consistency, focus visibility when shown, and whether framing supports the proof. Inspect source for semantic border roles and focus/error/disabled rules. Do not invent control states solely to demonstrate the edge system.

## Done when

- Every visible edge clarifies structure, state, or art direction
- Weight, color, and radius form a coherent vocabulary
- Focus treatment is source-complete for real controls
- Border and shadow do not compete
- Removing an edge would make the relevant relationship less clear
