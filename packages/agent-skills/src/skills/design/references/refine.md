# Refine

Use refine to change the character or resilience of an existing landing page without pretending it needs a new brief. Choose one coherent move from observed evidence and apply it only as broadly as the user requested.

## Choose the move

### Push

Use when the page is safe, timid, or forgettable. Increase commitment through stronger hierarchy, a clearer proof object, more decisive composition, more specific imagery, or a better-defined type and color voice. Do not translate “stronger” into more gradients, glass, noise, or animation.

### Settle

Use when the page is loud or exhausting. Reduce competing focal points, saturation, motion, depth, and repeated emphasis while preserving identity. Quiet means controlled, not gray or generic.

### Strip

Use when the path is obscured by filler. Remove repeated claims, decorative containers, redundant actions, weak sections, and effects that do not help the visitor decide, learn, explore, or compare. Stop if removal changes the product promise.

### Proof

Use when the composition fails under realistic content or page conditions. Test long names, wrapped headlines, missing images, translated copy, narrow screens, wide screens, reduced motion, and the real states of forms or demos that exist.

### Activate

Use when the visitor cannot reach the primary value or action. Improve the first viewport, proof sequence, CTA clarity, useful defaults, and contextual explanation. Do not add a product tour to a marketing page without a real need.

### Texture

Use when the page works but lacks memorable character. Add one or two specific material, type, image, or motion moments where emotion already belongs. Texture spread everywhere becomes noise.

### Technical ambition

Use when the brief genuinely benefits from a demanding visual or interaction idea. Name the benefit and risk first. Keep the effect bounded, accessible, responsive, and justified by the page premise.

## Scope boundary

A precise request such as “make the hero quieter” stays in the hero. A broad request such as “make the whole page more editorial” can apply one move across the full page. Do not combine opposing moves unless distinct zones demonstrably need different treatment.

Preserve exact content, working behavior, and established brand decisions outside the chosen move. Read any additional foundations required by the root skill before editing.

## Verification

Compare the affected rendered surface with the stated move. The change must be visible in character, clarity, resilience, or delight, not only in a small numeric CSS adjustment. Inspect source for semantics, responsive rules, focus, and reduced motion when touched.

## Done when

- The chosen move answers an observed problem
- The requested surface visibly changed in the intended direction
- The primary message, proof, and action are clearer or more resilient
- The page still belongs to the same brand and product
- No opposite corrective pass is now needed
- Every completion claim matches visible or source-inspected evidence
