# Color

Color sets atmosphere, hierarchy, and semantic meaning. Use recolor as a full mode when the palette is the request; when loaded as support, change only roles touched by the active mode.

## Start with the emotional arc

Define what visitors should feel at arrival, proof, decision, risk, and completion. Choose hue, lightness, chroma, and distribution to support that arc rather than decorating empty space.

Useful strategies include:

- **Whisper:** tinted neutrals with one scarce action color
- **Statement:** one brand hue owns a meaningful part of the surface
- **Conversation:** several named colors, each with a distinct job
- **Flood:** color becomes the environment for an earned brand moment

These are choices, not quotas. Distribution follows content and contrast rather than a fixed percentage formula.

## Build semantic roles

Define only roles the page actually uses:

- Canvas and elevated or alternate surfaces
- Primary and muted text
- Borders and dividers
- Primary and secondary actions
- Focus and selection
- Success, warning, and error for real states
- Disabled treatment for real controls
- Domain-specific data or status colors when a product demo contains them

Apply roles to actual elements. A declared palette with unused values is not a color system.

## Prefer perceptual control

For a new palette, OKLCH is a useful default because lightness changes behave more predictably. Keep chroma restrained near very light and very dark values, and tint neutrals toward the brand hue when it creates cohesion.

Existing brand colors remain authoritative. Adapt supporting values around them rather than “correcting” supplied identity without permission.

## Preserve hierarchy without hue

Primary, secondary, and muted elements should remain distinguishable by lightness, weight, shape, label, or position. Never make color the only signal for state or action.

Inspect contrast in source and rendered output. Body copy, muted copy, controls, focus indicators, and text over imagery all need stable separation. Use overlays or backed surfaces for image text rather than relying on a favorable crop.

## Dark surfaces

Treat dark presentation as an authored theme, not an inversion. Use surface lightness and restrained borders for depth; reduce glowing chroma; tune light text weight and leading. Do not add a dark theme unless the user requests it or the page already supports it.

## Refuse the category default

A palette should not be predictable from industry alone. Blue-purple is not automatic technology, teal is not automatic health, green is not automatic money, and navy is not automatic trust. Use the prompt's artifact, environment, materials, supplied imagery, or brand assets as evidence.

Avoid gradient text and glossy CTA gradients as substitutes for hierarchy. Gradients are valid when they express the page's physical or atmospheric premise and remain readable.

## Verification

Use screenshots to judge mood, action prominence, text legibility, image overlays, and cross-section consistency. Use source inspection to verify semantic reuse, focus colors, state redundancy, and theme/media rules. Do not claim a simulation that available tools did not run.

## Done when

- Every used color has a named role
- The primary action is clear without spreading accent everywhere
- Text and controls remain legible on their real surfaces
- State meaning survives without hue alone
- Neutrals, imagery, and brand colors feel related
- The palette belongs to this brief rather than the industry median
