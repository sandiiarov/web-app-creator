# Deslop

Use deslop to replace observed generated-design reflexes with specific decisions. It is treatment, not a second diagnosis and not a generic polish pass.

> **Manifest mutation lock:** Reading this operation file never opens the lock by itself. Reuse successful full reads already present in the project conversation, then complete every still-unread root-manifest path and receive its result before calling `edit` or `generate_image`.

The root skill requires smell, checkup, and review before deslop. Run those diagnostics in memory, preserve their evidence, and rank only findings observed on the current page. Before editing, read the foundation references implicated by those findings.

## Rank the treatment

Fix in this order:

1. Critical trust, readability, accessibility, or responsive failures
2. Prompt drift: wrong name, category, proof object, claim, or action
3. Structural sameness: center stacks, equal tile grids, formulaic section order
4. Identity sameness: category-default palette, default type, generic imagery
5. Empty proof: decorative mockups, unsupported statistics, vague claims
6. Unnecessary physical effects: blur, rails, shadows, pills, borders, motion
7. Small isolated tells

Do not fix an unobserved issue to make the pass feel comprehensive. Honor explicit user scope and preserve working behavior.

## Replace, do not repaint

For each finding:

1. Name the reflex that produced it.
2. Identify the prompt invariant or page job the replacement must serve.
3. Choose a specific structural, visual, verbal, or behavioral decision.
4. Apply it to the actual HTML.
5. Verify the old tell is gone and the replacement remains coherent with the rest of the page.

A smell is not fixed by swapping one fashionable default for another. A purple gradient becoming an orange gradient is still an unearned gradient. Equal cards becoming differently colored equal cards is still an unchosen hierarchy.

## Common treatment moves

- Replace a center stack with a composition driven by proof, reading path, or comparison.
- Break equal feature tiles by prioritizing one capability, changing format, or removing filler.
- Replace a category-default palette with color tied to a domain object, environment, or brand asset.
- Tune or replace default typography with a reason grounded in the page's physical or cultural lane.
- Remove accent rails, icon toppers, decorative glass, and unearned shadows; restore hierarchy with spacing, type, framing, or evidence.
- Replace number monuments with contextual proof: before/after, workflow evidence, customer outcome, or a metric with source and meaning.
- Remove repeated bounce, fade-up cascades, and motion that explains nothing.
- Rewrite vague claims and CTA labels so they name the offer, evidence, and action.

## Cohesion pass

After treatment, inspect the full affected surface. Structural fixes can expose seams between sections; color or type changes can invalidate old tokens; removed effects can reveal weak spacing. Resolve those consequences without expanding into unrelated redesign.

## Verification

Use screenshots for visible before/after judgment and source inspection for semantics, responsive rules, focus, and reduced motion. Re-run the smell judgment against the rendered result. If a remaining tell is outside scope or cannot be verified, say so rather than claiming it was removed.

## Done when

- Every treated item maps to an observed diagnostic finding
- Prompt drift and critical failures were handled before decorative tells
- Replacements belong to this product rather than another template lane
- No treatment created a new dominant smell
- The page still performs its original job and preserves valid behavior
