# Prompt Panel Package DOX

## Purpose

- Owns landing-page prompt panel UI + landing conversation domain model it renders. Source-consumed by `@workspace/client` (and future consumers) via `@workspace/prompt-panel`.

## Ownership

- `src/prompt-panel.tsx`: root `PromptPanel` component, its `PromptPanelProps`, horizontal-resize `ResizeState`/handlers, `PanelResizeHandle` edge handles.
- `src/`: panel UI components (`panel-header`, `panel-body`, `panel-command-menu`, `composer`, `turn-message`, `spend-popover`, `turn-steps`, `streamdown-content`, `model-dropdown`, `status-pill`, `chat-empty-state`, icons) + helpers (`panel-constants`, `panel-status`, `panel-storage`, `model-role-meta`, `spend-summary`, `format`, `keyboard-shortcut`).
- `src/panel-storage.ts`: read-only panel position/layout persistence — owns `landing.promptPanel.position.v1` localStorage key, `StoredPanelState` (`x`/`y`, `collapsed`, `layout`, `width`), `readStoredPanelState`, public `readStoredPanelLayout` (effective layout; treats collapsed panel as `floating` so minimized panel reserves no docked column) + `readStoredPanelWidth`. Component keeps write logic (`writeStoredPanelState`) — intertwined w/ docked-side derivation.
- `src/domain.ts`: landing conversation domain model — model options (text `LANDING_MODEL_OPTIONS`, + `LANDING_VISION_MODEL_OPTIONS` + `LANDING_IMAGE_MODEL_OPTIONS`), per-category `LandingModels` selection + `DEFAULT_LANDING_MODELS` + `resolveLandingModels`, attachment types, conversation model (`LandingTurn`, incl durable `stopped` terminal state, `TurnPart` variants, optional tool-call arg images, `RetryPart` inlined), cost/usage types, `LandingAgentSendInput`, formatting helpers.
- `src/keyboard-shortcuts.ts`: `KEYBOARD_SHORTCUTS` metadata + types.
- `src/index.ts`: public barrel (`PromptPanel`, `PromptPanelProps`, `readStoredPanelLayout`, `readStoredPanelWidth`, `PANEL_WIDTH`, `MIN_PANEL_WIDTH`, `maxPanelWidth`, `clampPanelWidth`, `DEFAULT_PREVIEW_VIEWPORT`, `PREVIEW_VIEWPORTS`, domain model, keyboard-shortcuts, panel-constants types incl `PreviewViewport`).

## Local Contracts

- Source-consumed like `@workspace/ui`: `exports` point at `./src/*`, no build step, consumers import through package exports (`@workspace/prompt-panel`, `@workspace/prompt-panel/domain`, `@workspace/prompt-panel/keyboard-shortcuts`).
- Depends on `@workspace/ui` + catalog deps (`lucide-react`, `react-hotkeys-hook`, `streamdown`, `@streamdown/code`); `react`/`react-dom` are peer deps. Tailwind design tokens from `packages/ui/src/styles/globals.css` (referenced by this pkg's oxlint/oxfmt configs).
- Package owns **domain model only**. Must not import app code, reference `import.meta.env`, or contain SSE/transport logic. `RetryPart` defined inline (not extending app's wire `RetryEvent`) to avoid package→app dependency.
- App behavior injected via props, never imported: navigation via `onAllProjects`, theme via `theme: PanelTheme` + `onToggleTheme`, current effective layout via optional `onLayoutChange?: (layout: PanelLayout) => void` (reports `floating`/`left-sidebar`/`right-sidebar`; collapsed panel reports `floating`), selected preview viewport via controlled pair `viewport: PreviewViewport` + `onViewportChange` (consumer-owned; not persisted by panel). Opt-in `onReloadPreview?: () => void` renders header Refresh button that asks consumer to reload preview iframe; disabled until consumer reports downloadable HTML (`canDownload`). `readStoredPanelLayout` lets consumer init its own preview layout from persisted state w/o first-paint flash. Package imports no `react-router-dom` and no theme context.
- Dropdown-menu triggers in `src/panel-command-menu.tsx` (`PanelLayoutMenu`, `PreviewViewportMenu`, `PanelSettingsMenu`) wrap `DropdownMenuTrigger` in Radix `Tooltip`. Each menu controls own `open` state (`PanelLayoutMenu` via parent-controlled `open` prop; others via local `useState`) and binds `Tooltip open={open ? false : undefined}` so tooltip forced closed while menu open — w/o this tooltip lingers/overlaps menu content, since pointer stays hovered on stationary trigger and `TooltipProvider` uses `delayDuration={0}`. `PreviewViewportMenu`'s trigger icon reflects current selection (`Monitor`/`Tablet`/`Smartphone`); choices render as plain menu items, no checkbox/radio indicators. Reuse this pattern for any new header dropdown w/ tooltip on trigger.
- `PanelTheme = 'dark' | 'light' | 'system'` lives in `src/panel-constants.ts`, alongside `PreviewViewport` (`'desktop' | 'mobile' | 'tablet'`), ordered `PREVIEW_VIEWPORTS` list, `DEFAULT_PREVIEW_VIEWPORT` (`'desktop'`), `MIN_PANEL_WIDTH` (default `PANEL_WIDTH` equals min), `maxPanelWidth()` (half viewport, the max), `clampPanelWidth(width, max?)`, `PANEL_WIDTH_CSS_VAR` (`--landing-panel-width` custom-property name).
- Panel horizontally resizable from both edges. Rendered width **is** `--landing-panel-width` CSS custom property, registered as typed `@property` in `packages/ui` globals (CSS `initial-value` default, so no JS needed for first paint; can be `transition`ed later) — panel never sets width from React inline value. Numeric width in ref (no React width state, no re-render during gesture); `setPanelWidthVar` writes var on `:root` on mount + while resizing; width persists via `writeStoredPanelState`. Resize mirrors position drag: pointer-captured on stable gesture element (`currentTarget`, never volatile `event.target` child), rAF-throttled, direct-DOM writes during gesture, finalized on `pointerup` + `onLostPointerCapture`/`onPointerCancel` so lost capture (mid-gesture re-render, OS interruption, or cursor leaving window) never strands gesture; both finalizers idempotent (drag tolerates second call; resize returns early once `ResizeState` cleared). Right-edge handle pins panel left; left-edge handle pins panel right (and repositions `left`); each clamped to `[MIN_PANEL_WIDTH, maxPanelWidth()]` + available viewport so panel cannot overflow. Section carries `data-landing-prompt-panel` and, while width resize active, `data-resizing`. Its `height` always CSS-transitioned (only changes on dock/undock/collapse, never mid-gesture); editor disables preview-offset transition under `:has([data-resizing])` (see `packages/ui` globals) so margin tracks `--landing-panel-width` exactly during resize.
- Cost formatting inside popover renders USD w/ exactly 4 digits after decimal point for zero, tiny, larger costs; do not use less-than-cent shorthand there. Icon-free spend trigger displays only dollar amount rounded to 2 decimals. Conversation no longer renders per-turn Spend / tokens blocks. `SpendPopover` sits immediately after model selector in composer, aggregates latest `stats` snapshot from every turn (incl active turn), updates cumulative project cost, cost categories, token counts whenever rolling stats arrive; token breakdown shows Input, Output, Cached (zero when unavailable) w/o Reasoning row; legacy stats w/o breakdown count full cost as LLM spend. Cost-category rows reuse `MODEL_ROLE_META` so Text/LLM, Image, Vision icons + colors match model dropdown; Scrape uses local Firecrawl brand mark in specified OKLCH orange. LLM, image gen, vision OCR, scrape, Input, Output, Cached rows stay visible w/ zero values when project has no matching usage.
- `TurnToolBlock` renders validated `http(s)`, root-relative, or supported raster data-URL images inside expanded Args section. Analyze-image previews from client-only attachment payload; screenshot previews from server's persisted screenshot URL. Keep image bytes out of durable JSON logs.

## Work Guidance

- Keep `PromptPanel` presentational: state belongs in hooks/consumers; panel receives data + callbacks. Theme + navigation passed in from composition site.
- When domain model changes, update `src/domain.ts` + re-export from `src/index.ts`; transport/wire shapes stay in consuming app.
- `src/model-dropdown.tsx` renders one outline trigger showing all 3 role selections side by side (text, image, vision) as two-icon segments — lucide role icon + selected model's provider logo — separated by dividers, each w/ tooltip showing `[provider logo] model name` (relies on app-level TooltipProvider). Opens menu whose top is segmented toggle (`[Icon] Text | [Icon] Image | [Icon] Vision`, lucide role icons) that switches which role's single radio list shows below. Only active role's model list renders. Every Text option uses OpenRouter's `:nitro` routing variant; `resolveLandingModels` migrates restored unsuffixed/other-variant text selections to `:nitro`; Image + Vision ids unchanged. Brand icons map by model id in `MODEL_ICONS`; add per-id entry as new provider logos land. Default Vision option is `bytedance-seed/seed-2.0-mini` and uses existing `BytedanceIcon`.
