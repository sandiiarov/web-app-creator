## Why this matters

`pnpm audit --prod` reports a **HIGH**-severity advisory for `vite` in the
installed range `>=8.0.0 <=8.0.15`; vite is a direct catalog dependency
(`pnpm-workspace.yaml`) and flows through `@workspace/vite-config` /
`@tailwindcss/vite` into the client **build and dev-server path**. The
installed version is `8.0.14`; the patched version is `>=8.0.16` (8.0.16
exists on the npm registry and is the latest 8.0.x). Bumping the catalog
floor to `^8.0.16` both clears the advisory and prevents a future
`pnpm install` from regressing into the vulnerable range. Real-world runtime
risk is low (loopback / operator-controlled single-tenant tool), but the fix
is a one-line catalog edit + lockfile refresh, and CI runs `pnpm audit`-style
hygiene via the build chain, so this is cheap insurance.

## Steps

### Step 1: Raise the catalog floor

In `pnpm-workspace.yaml`, change line 40 from `vite: "^8.0.14"` to
`vite: "^8.0.16"` (keep the 2-space indent and double quotes — match the
surrounding catalog entries exactly).

**Verify**: `grep -n 'vite:' pnpm-workspace.yaml` → the line reads
`  vite: "^8.0.16"` (and `@vitest/coverage-v8` / `vitest` lines are
untouched).

### Step 2: Refresh the lockfile

Run `pnpm install`. Because `8.0.14` no longer satisfies `^8.0.16`, pnpm is
forced to resolve `8.0.16` and rewrite `pnpm-lock.yaml`.

**Verify**: `node -p "require('./node_modules/vite/package.json').version"`
→ prints `8.0.16`.

### Step 3: Confirm the CI lockfile gate still passes

**Verify**: `pnpm install --frozen-lockfile` → exit 0 (proves the regenerated
lockfile is in sync with the manifests — this is the exact gate CI runs).

### Step 4: Confirm the advisory cleared and the build still works

**Verify**:
- `pnpm audit --prod 2>&1 | grep -i vite` → no output (vite no longer listed
  under any severity). Run `pnpm audit --prod 2>&1 | tail -3` and confirm the
  summary dropped by the vite HIGH (the totals line should show one fewer
  HIGH than before).
- `pnpm --filter @workspace/client build` → exit 0 (vite is the client build
  tool; this proves the bump did not break the build or the tailwindcss vite
  plugin).
