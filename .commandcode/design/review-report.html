// PRIORITY ISSUES
P1

Status chip occupies the disclosure corner

The checkmark appears in a bordered square and has more visual weight than the adjacent chevron. It reads as the primary control even though expansion is the important interaction.

FIX: Remove the standalone status chip. Keep the right edge for one chevron. Move state meaning into border/background/icon color.
P1

Done state is too loud

Completed diagnostic rows are not the main moment. A checkmark on every completed row creates checklist noise in long agent runs.

FIX: Let done be neutral: normal border, normal background, tool icon, intent, chevron.
P2

Loading state should avoid right-edge churn

Fast LLM responses already make diagnostic rows appear quickly. A heavy spinner/status chip would make the row feel busier than necessary.

FIX: Prefer amber/yellow border, faint background, and icon tint. If a spinner remains, do not put it where disclosure belongs.
